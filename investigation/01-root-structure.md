## Item-by-Item Analysis

### `.agents/` — KEEP
Agent skill definitions for OpenCode's own development. Contains `skills/` subdirectory with SKILL.md files for custom agent capabilities used when running OpenCode on itself.

### `.opencode/` — KEEP
Self-referential OpenCode configuration. Contains:
- `opencode.jsonc` — Provider, permission, and tool config
- `agent/` — Built-in agent definitions (docs, translator, triage, duplicate-pr)
- `command/` — Custom CLI commands
- `glossary/` — Domain terminology
- `themes/` — UI themes
- `tool/` — Custom tool definitions

This is the config that OpenCode uses when you run it on this repo itself.

### `.husky/` — KEEP
Git hooks for code quality (pre-commit linting, etc.). Standard development tooling.

### `.turbo/` — KEEP (auto-managed)
Turborepo build cache. Auto-generated, never edit manually.

### `.venv/` — KEEP (local only)
Python virtual environment for local tooling scripts. Not committed to git.

### `AGENTS.md` — KEEP
Documentation explaining how AI agents (like Claude Code) should work with this repo. Has been modified from the original upstream version.

### `CONTRIBUTING.md` / `LICENSE` / `README.md` / `SECURITY.md` — KEEP
Standard project documentation.

### `STATS.md` — KEEP
Project statistics file. Auto-generated by `script/stats.ts`.

### `bun.lock` / `bunfig.toml` / `package.json` — KEEP (essential)
Core monorepo configuration files. `package.json` defines workspaces and the shared dependency catalog.

**Workspaces currently defined (already clean):**
```json
"packages": [
  "packages/opencode",
  "packages/plugin",
  "packages/util",
  "packages/script",
  "packages/sdk/js"
]
```

### `turbo.json` — KEEP (already clean)
Turborepo pipeline. Only references generic tasks (`typecheck`, `build`, `opencode#test`) — no stale package references.

### `tsconfig.json` — KEEP
Root TypeScript config. Extended by individual packages.

### `external/` — INVESTIGATE / REMOVE
Currently contains only `external/mistral-vibe/` which appears to be an empty placeholder. Not referenced by any package. **Can be removed.**

### `install` — KEEP
Shell script for end-user installation of the CLI. Used in the public installation flow.

### `patches/` — KEEP (required)
Three dependency patches required for compatibility:
- `@ai-sdk/xai@2.0.51.patch` — Fix for xAI provider
- `@openrouter/ai-sdk-provider@1.5.4.patch` — Fix for OpenRouter provider
- `solid-js@1.9.10.patch` — Fix for SolidJS (even though SolidJS packages are removed, check if still referenced)

### `script/` — KEEP
Release automation scripts:
- `beta.ts` — Publish beta releases
- `changelog.ts` — Generate CHANGELOG
- `format.ts` — Code formatting utilities
- `generate.ts` — Code generation
- `hooks/` — Release hooks
- `publish.ts` — Publish to npm
- `release/` — Release workflow
- `stats.ts` — Generate project stats
- `version.ts` — Version management

### `specs/` — KEEP
Contains `project.md` — the REST API specification for multi-project session management. Architectural reference.
